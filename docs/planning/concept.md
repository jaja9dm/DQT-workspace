## 1. 프로젝트 개요

### 목적
퀀트 투자란 **감이 아닌 숫자와 규칙으로 하는 투자**다.
본 시스템은 Claude API를 두뇌로, KIS API를 실행 엔진으로 사용하여
사람의 감정 개입 없이 매일 자동으로 매수/매도 판단을 내리는 시스템을 구축한다.

### 핵심 철학

**감정 배제**
공포에 팔고 탐욕에 사는 것은 인간의 본능이다. 본 시스템은 시장이 흔들려도 사전에 정의된 규칙만 실행한다.

**규칙 기반**
명확한 조건이 충족될 때만 거래한다. 애매한 상황에서는 홀드(hold)가 기본값이다.

**이성적 손절 — 기회비용의 최대화**
손절은 실패가 아니라 **자본을 더 나은 기회로 이동시키는 의사결정**이다.
손실 -5%에서 손절하면 원금 회복에 5.3%만 필요하지만, -30%까지 버티면 42.9%를 벌어야 원금이다.

**큰 수의 법칙 — 51 대 49의 게임**
단 한 번의 거래로 승부를 보려 하지 않는다.
핵심은 **올바른 확률 구조를 충분히 많이 반복하는 것**이다.

**3종목 집중 원칙**
시장이 좋든 나쁘든 하루 최대 매수 종목은 3개다. 분산이 아닌 집중으로 확신 있는 종목에만 진입한다.
각 종목은 역할(슬롯)이 다르다 — 주도주, 돌파매매, 눌림목매매.

**완전 당일 청산 원칙**
오버나잇을 하지 않는다. 모든 포지션은 당일 14:50 이후 수익 확정 또는 15:20까지 강제 청산한다.
수익이 나더라도 다음 날의 리스크를 안고 가지 않는다.

**자기학습 피드백**
매일 거래 결과를 자동 분석하여 진입 점수 파라미터를 튜닝한다.
Claude가 오늘의 패턴을 복기하고, 내일의 전략을 제안한다.

## 2. 핵심 거래 철학 — 슬롯 시스템

### 3슬롯 구조

매일 아침 9시, 시장이 열리면 3개의 슬롯을 채운다. 각 슬롯은 독립적이다.

| 슬롯 | 역할 | 선정 기준 |
|------|------|-----------|
| 👑 **leader** | 오늘의 주도주 | 전체 시장 거래대금 1위권 + 수급(외인/기관) + 주도 섹터 대장 |
| 🚀 **breakout** | 돌파매매 | 갭업 +3% 이상 or 볼린저밴드 상단 돌파 + OBV 상승 |
| 🔄 **pullback** | 눌림목매매 | 오늘 -1%~-5% 하락 중 장중 반등 시작 + OBV 상승 + RSI ≤ 75 |

### 타임라인

```
09:00 ─── 장 개시 (관찰 모드 시작)
           ↓ 개장 초 변동성 안정 대기
           ↓ 3분봉 2개 + 5분봉 1개 완성 대기
09:07 ─── 슬롯 탐색 시작 (Claude 분석 → 슬롯 배정)
           ↓ 5분마다 스캔 반복
           ↓ 슬롯이 채워지지 않으면 계속 탐색
11:30 ─── 슬롯 탐색 마감 (신규 슬롯 배정 중단)
           ↓ 이후는 기존 포지션 관리에만 집중
14:50 ─── 수익권 포지션 전량 익절 시작
15:20 ─── 손익 무관 전체 강제 청산
15:35 ─── 실시간 엔진 정지
```

### 슬롯 독립성

슬롯 2(breakout)가 이상해도 슬롯 1과 3은 정상 매매한다.
문제가 생긴 슬롯만 교체를 탐색하고, 나머지 슬롯은 영향 없다.

포지션 청산 시 해당 슬롯은 자동으로 `empty` 상태가 되며, 11:30 이전이라면 새 종목 탐색을 재개한다.

## 4. 팀별 구성 및 역할

### 공통 인프라 0-1. KIS 게이트웨이

**한 줄 목표:** KIS API 토큰과 Rate Limit를 한 곳에서 관리하여 토큰 충돌·만료 장애를 원천 차단한다.

| 기능 | 내용 |
|------|------|
| 토큰 관리 | Access Token 발급·갱신·만료 감지. 만료 30분 전 자동 갱신 |
| Rate Limit | 초당·분당 한도 큐잉. 우선순위: TRADING > REALTIME > BACKGROUND |
| 요청 라우팅 | 포지션 감시 > 매매팀 > 수집팀 순 우선 처리 |
| 장애 처리 | 3회 재시도 → 실패 시 전 팀 알림 + 거래 중지 |
| 모의/실전 | `KIS_MODE=paper|live` 환경변수 1개로 전체 전환 |

---

### 공통 인프라 0-2. 주도주 감지 (`market_leaders.py`)

**한 줄 목표:** "오늘 시장에서 돈이 가장 많이 몰리는 종목"을 실시간으로 파악한다. 우리가 스캔하는 450종목 안에 없어도 찾아낸다.

**데이터 소스:** KIS 거래대금 순위 API (`FHPST01730000`)

**동작 방식**
```
KOSPI 거래대금 TOP30 + KOSDAQ 거래대금 TOP30
        ↓ 합산 후 거래대금 내림차순 정렬
거래대금 TOP30 확정
        ↓ 필터: 등락률 > 0 AND (외인 OR 기관 순매수)
주도주 확정 (통과 없으면 TOP5 그대로 사용)
        ↓
5분 캐시 → Claude 프롬프트에 삽입 + 슬롯 leader 선정 우선 활용
```

**제공 함수**
- `get_leaders(n)` — 거래대금+수급 기준 주도주 상위 n개
- `get_leader_tickers()` — 주도주 ticker 집합 (O(1) 조회)
- `get_leader_context_str()` — Claude 프롬프트 삽입용 문자열

### 공통 인프라 0-3. 섹터 로테이션 캐시 (`sector_rotation.py`)

**한 줄 목표:** 오늘 어느 섹터에 돈이 몰리는지 실시간으로 추적한다.

**동작 방식**
```
스캔 사이클마다 전체 종목의 등락률을 섹터별 집계
        ↓ 섹터별 평균 수익률 vs KOSPI 수익률 비교
강세 섹터 (vs_kospi > +0.5%) → 🔥 태그
약세 섹터 (vs_kospi < -0.5%) → 🧊 태그
        ↓
sector_strength 테이블에 저장 + Claude 프롬프트에 삽입
```

**활용**
- leader 슬롯: 강세 섹터 대장주 우선 선정
- breakout 슬롯: 강세 섹터 내 돌파 종목 우선
- 동일 섹터 약한 포지션 자동 교체 (position_monitor)

### 팀 1-1. 국내 주식팀

**한 줄 목표:** 매일 아침 9:07부터 3개의 슬롯(leader/breakout/pullback)에 종목을 채운다.

**타임라인**

| 시각 | 동작 |
|------|------|
| 09:00 | 스캔 시작, 주도주 갱신, Gate 체크 |
| 09:00~09:07 | **관찰 모드** — 슬롯 배정 없이 데이터만 수집 |
| 09:07 | 슬롯 탐색 시작 — Claude에게 슬롯별 종목 1개씩 요청 |
| 매 5분 | 빈 슬롯만 재탐색 (이미 배정된 슬롯은 스킵) |
| 11:30 | 탐색 마감 — 신규 슬롯 배정 중단 |

**Gate 사전 체크 (API 비용 최적화)**

| Gate | 조건 | 결과 |
|------|------|------|
| Gate 1 | 리스크 레벨 ≥ 4 | 전체 스킵 |
| Gate 2 | 글로벌 outlook == `negative` | 전체 스킵 |
| Gate 3 | 국내 시황 점수 < -0.3 | 전체 스킵 |

**Claude 출력 형식 (슬롯별)**
```json
{
  "slots": {
    "leader":   {"ticker": "005930", "signal_type": "sector_momentum", "reason": "반도체 주도섹터 대장·외인+기관 동시매수"},
    "breakout": {"ticker": "247540", "signal_type": "gap_up_breakout",  "reason": "갭업+3.2%·볼밴상단돌파·OBV↑"},
    "pullback": {"ticker": "035720", "signal_type": "pullback_rebound", "reason": "눌림-2.1%·장중반등+0.8%·OBV↑"}
  }
}
```

**DB 저장**
- `hot_list` 테이블: slot 컬럼 포함하여 저장
- `slot_assignments` 테이블: 오늘 날짜별 슬롯 상태 관리 (active/empty)

**즉시 트리거**
- 거래량 5배 이상 급등 → 정규 주기 무시하고 즉시 재스캔
- 가격 +5% 이상 급등 → 즉시 경보

### 팀 3. 매매팀

**한 줄 목표:** 슬롯에 배정된 종목에 대해 5단계 게이트를 통과하면 분할 매수를 실행한다.

**5단계 게이트 (순서대로)**

```
Gate 1 — 리스크 레벨 ≥ 4 → 즉시 중단
Gate 2 — 글로벌 outlook negative → 매수 보류
Gate 3 — 국내 시황 점수 < -0.3 → 매수 보류
Gate 4 — 종목별 필터 (RSI, 거래량, OBV, ATR, 외인/기관 수급)
Gate 4.2 — 진입 신뢰도 점수 ≥ 50점 (자기학습 파라미터)
Gate 5 — Claude 최종 판단 (슬롯 역할·시황·기술 지표 종합)
```

**진입 신뢰도 점수 구성 (Gate 4.2)**

| 항목 | 배점 |
|------|------|
| 외인 + 기관 동시 순매수 | +8pt |
| 외인 단독 순매수 | +4pt |
| 기관 단독 순매수 | +3pt |
| 외인 + 기관 동시 순매도 | -5pt |
| OBV 상승 기울기 | +3pt |
| 주도섹터 소속 | +5pt (sector_hot_bonus) |
| 약세섹터 소속 | -3pt (sector_cold_penalty) |
| 시장 주도주 | +4pt |
| MACD Pre-Cross 감지 | +5pt |
| RSI 과열 (> 72) | 포지션 50% 감축 |
| RS 하락장 hard fail | 진입 차단 |

**Hard Fail 조건 (즉시 차단)**

| 조건 | 설명 |
|------|------|
| RSI > 82 (일반 신호) | 극단 과열 차단 |
| RSI > 95 (갭업/섹터급등 + OBV↑) | 테마 급등 시 완화 허용 |
| breakout 신호 + RSI < 55 | BB 상단 돌파인데 모멘텀 미확인 (가짜 신호 차단) |
| OBV 역행 + RSI > 70 | 수급 없는 상승 (갭업/섹터급등 면제) |
| StochRSI > 88 | 단기 극과매수 (갭업/섹터급등 면제) |
| 체결강도 < 75 | 매도세 압도 |
| 갭업 + BB폭 미확대 | 가짜 브레이크아웃 |

**분할 매수 정책**

```
1차: 신호 발생 즉시 — 목표 수량의 40%
2차: 1차 체결 후 추가 확인 시 — 목표 수량의 35%
3차: 포지션 강화 시 — 목표 수량의 25%
```

**슬롯별 전략 특성**

| 슬롯 | 손절 기준 | 익절 방식 | 특이사항 |
|------|----------|----------|----------|
| leader | ATR 기반 동적 손절 | 트레일링 스톱 전면 적용 | MACD bearish 시 즉시 1/3 매도 |
| breakout | ATR 기반 동적 손절 | 트레일링 스톱 | 갭업 포지션 특성 상 손절선 넓게 |
| pullback | ATR 기반 동적 손절 | 트레일링 스톱 | 눌림목 반등 확인 후 진입 |

### 팀 4-1. 포지션 감시

**한 줄 목표:** 매수 후 보유 포지션을 계속 지켜보며 손절·익절·타임컷 조건 도달 시 즉시 주문을 실행하고 슬롯을 자동 해제한다.

**감시 주기:** 90초 (WebSocket 1차 + 거래소 지정가 2차 + 폴링 3차 3중 안전망)

**감시 및 실행 조건**

```
[트레일링 스톱] — 기본 손절 방식
  초기 손절선 = 매수가 × (1 - ATR_PCT × 1.5)   ← ATR 기반 동적 산출
  trigger_pct 수익 달성 시 손절선 상향 시작
  손절선은 절대 내려가지 않음 (단방향 상승)
  현재가 ≤ 손절선 → 전량 시장가 매도

[사다리 매수]
  현재가 ≤ 매수가 × (1 - 20%) 이고 미실행 시 → 추가 매수

[MACD 분할 익절]
  MACD bearish + 수익 +5% 이상 → 보유량 1/3 매도
  MACD sell_prep (5분봉 히스토그램 피크 꺾임) → 손절선 절반으로 타이트

[동일 섹터 약포지션 교체]
  같은 섹터의 더 강한 Hot List 종목 존재 + 현재 포지션 수익 < 1.5% → 교체 매도

[타임컷]
  보유 5 영업일 초과 → 전량 청산

[장마감 강제청산]
  14:50~15:20: 수익권이면 전량 익절
  15:20 이후: 손익 무관 전량 강제 청산

[긴급 전량 청산]
  리스크 레벨 5 → 전체 포지션 즉시 청산
```

**슬롯 자동 해제**

포지션 청산 완료 시 → `slot_assignments` 테이블의 해당 슬롯을 `empty`로 업데이트 → 11:30 이전이면 국내 주식팀이 다음 사이클에서 해당 슬롯을 재탐색.

## 6. DB 핵심 테이블

```sql
-- 슬롯 배정 현황 (당일 기준)
CREATE TABLE slot_assignments (
  slot        TEXT PRIMARY KEY,  -- 'leader' | 'breakout' | 'pullback'
  ticker      TEXT,
  name        TEXT,
  signal_type TEXT,
  reason      TEXT,
  trade_date  DATE NOT NULL,
  status      TEXT DEFAULT 'empty',  -- 'active' | 'empty'
  assigned_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Hot List (국내 주식팀 출력)
CREATE TABLE hot_list (
  id               INTEGER PRIMARY KEY,
  ticker           TEXT,    name TEXT,    signal_type TEXT,
  volume_ratio     REAL,    price_change_pct REAL,
  rsi              REAL,    sector TEXT,  reason TEXT,
  momentum_score   REAL,    obv_slope REAL,   day_range_pos REAL,
  stoch_rsi        REAL,    bb_width_ratio REAL,
  trading_value    INTEGER, exec_strength REAL,
  rs_daily         REAL,    rs_5d REAL,
  frgn_net_buy     INTEGER, inst_net_buy INTEGER,
  atr_pct          REAL,
  slot             TEXT,    -- 'leader' | 'breakout' | 'pullback'
  created_at       DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 진입 컨텍스트 (자기학습 피드백)
CREATE TABLE trade_context (
  trade_id       INTEGER PRIMARY KEY,
  ticker         TEXT,    trade_date DATE,
  signal_type    TEXT,    rsi REAL,    entry_score REAL,
  momentum_score REAL,    rs_daily REAL,  rs_5d REAL,
  sector         TEXT,    exec_strength REAL,
  ob_imbalance   REAL,    entry_hhmm TEXT,
  created_at     DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 자기학습 파라미터
CREATE TABLE strategy_params (
  param_name   TEXT PRIMARY KEY,
  current_val  REAL,    default_val REAL,
  min_val      REAL,    max_val REAL,
  description  TEXT,    tuned_by TEXT,
  updated_at   DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 섹터 강도
CREATE TABLE sector_strength (
  sector      TEXT PRIMARY KEY,
  avg_ret_1d  REAL,    vs_kospi REAL,
  stock_count INTEGER,
  updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 종목별 누적 패턴 통계
CREATE TABLE ticker_stats (
  ticker            TEXT PRIMARY KEY,
  name              TEXT,
  total_trades      INTEGER,  win_count INTEGER,  loss_count INTEGER,
  win_rate          REAL,     avg_pnl_pct REAL,
  avg_win_pct       REAL,     avg_loss_pct REAL,
  avg_hold_minutes  REAL,     best_entry_hour INTEGER,
  frgn_buy_win_rate REAL,     inst_buy_win_rate REAL,
  best_signal_type  TEXT,     notes TEXT,
  last_updated      DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 7. 리스크 정책

- **3종목 상한:** 하루 최대 매수 3종목 (슬롯 수와 동일)
- **당일 청산:** 오버나잇 없음. 14:50부터 익절 시작, 15:20 강제 청산
- **분할 매수:** 단일 진입 금액은 전체 자산의 10% 이하
- **일일 최대 손실:** 전체 자산의 3% → 자동 거래 중지
- **ATR 기반 손절:** 종목 변동성에 따라 손절선 동적 산출 (고정 % 불사용)
- **실전 전환 조건:** 모의투자 3개월 + 수익률 플러스 확인
