## 1. 프로젝트 개요

### 목적
퀀트 투자란 **감이 아닌 숫자와 규칙으로 하는 투자**다.
본 시스템은 Claude API를 두뇌로, KIS API를 실행 엔진으로 사용하여
사람의 감정 개입 없이 매일 자동으로 매수/매도 판단을 내리는 시스템을 구축한다.

### 핵심 철학
- 감정 배제 — 공포에 팔고 탐욕에 사는 실수 제거
- 규칙 기반 — 명확한 조건이 충족될 때만 거래
- 검증 우선 — 실전 투자 전 반드시 모의투자 3개월 검증
- 리스크 관리 — 1회 주문 금액 상한선, 일일 최대 주문 횟수 제한

### 투자 주기
**데일리 트레이딩** — 매일 장 시작 전 분석, 장 시작 시 자동 주문

## 3. 단계별 상세 설명

### ① 데이터 수집 레이어

| 데이터 종류 | 소스 | 수집 방법 | 용도 |
|------------|------|----------|------|
| 뉴스 / 공시 | 네이버금융, KIND | 크롤링 / RSS | Claude 감성 분석 입력 |
| 주가 데이터 | FinanceDataReader | Python 라이브러리 | 기술적 지표 계산 |
| 재무 데이터 | FinanceDataReader | Python 라이브러리 | PER, PBR, ROE 팩터 |
| 기술적 지표 | 자체 계산 | pandas-ta | RSI, MACD, 볼린저밴드 |
| 시장 분위기 | 한국거래소, 증권사 | API / 크롤링 | 외국인 수급, 시장 전체 흐름 |

### ② Claude API 분석 레이어

Claude는 이 시스템의 **두뇌** 역할을 담당한다.
단순한 규칙 기반 로직이 아니라, 비정형 텍스트(뉴스, 공시)를 이해하고
복합적인 시장 상황을 종합 판단하는 데 활용한다.

**A. 뉴스 감성 분석**
- 입력: 종목명 + 뉴스 텍스트
- 출력: { "score": 0.85, "direction": "positive", "reason": "..." }
- 뉴스를 -1.0 ~ +1.0 감성 점수로 변환, 임계값 초과 시 매수 신호 활용

**B. 종목 매매 판단**
- 입력: 주가 데이터 + 기술적 지표 + 감성 점수 + 시장 분위기
- 출력: { "action": "buy", "ticker": "005930", "quantity": 10, "reason": "..." }
- 여러 지표를 종합하여 최종 매매 방향 결정

**C. 리스크 체크**
- 입력: 현재 포지션 + 매매 신호 + 시장 변동성
- 출력: { "approved": true/false, "reason": "...", "adjusted_quantity": 5 }
- 신호가 생성되더라도 리스크 과도 시 수량 조정 또는 거부

### ③ 전략 판단 레이어

초기 채택 전략 (우선순위 순):

1. **뉴스 감성 + 기술적 지표 복합 전략**
   - Claude 감성 점수 > 0.6 AND RSI < 40 → 매수
   - Claude 감성 점수 < -0.6 AND RSI > 70 → 매도

2. **거래량 급증 포착**
   - 평균 거래량 대비 3배 이상 → 모멘텀 추종

3. **전일 고점/저점 돌파**
   - 전일 고점 돌파 시 매수
   - 전일 저점 이탈 시 손절

### ④ 주문 실행 레이어 — KIS API

| 항목 | 내용 |
|------|------|
| 증권사 | 한국투자증권 |
| API | KIS Developers (REST 방식) |
| 지원 OS | Mac / Linux / Windows |
| 모의계좌 | 지원 (실전 전 반드시 사용) |
| 주문 방식 | 지정가 / 시장가 |
| 인증 | OAuth 2.0 토큰 방식 |

### ⑤ 피드백 레이어

- 매일 장 마감 후 체결 결과 DB 저장
- 주간 / 월간 수익률 리포트 자동 생성
- 전략별 성과 비교 → 우수 전략 가중치 증가
- 카카오톡 또는 Slack으로 일일 결과 알림

## 4. 기술 스택

| 분류 | 기술 | 용도 |
|------|------|------|
| 언어 | Python 3.11+ | 전체 시스템 |
| AI | Claude API (claude-sonnet-4) | 분석 / 판단 엔진 |
| 데이터 | FinanceDataReader, pandas | 주가 데이터 수집 및 처리 |
| 지표 | pandas-ta | 기술적 지표 계산 |
| 거래 | KIS API (한국투자증권) | 실제 주문 실행 |
| DB | SQLite → 추후 PostgreSQL | 거래 이력 / 로그 저장 |
| 스케줄러 | APScheduler / cron | 매일 자동 실행 |
| 알림 | 카카오톡 API / Slack Webhook | 결과 알림 |
| 버전관리 | Git / GitHub | 코드 및 문서 관리 |

## 5. 리스크 정책 (초안)

- 1회 최대 주문 금액: 전체 자산의 10% 이하
- 일일 최대 주문 횟수: 10회 이하
- 일일 최대 손실 한도: 전체 자산의 3% → 자동 거래 중지
- 종목 집중도: 단일 종목 최대 비중 20%
- 실전 전환 조건: 모의투자 3개월 + 수익률 플러스 확인
