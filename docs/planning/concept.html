1. 프로젝트 개요

퀀트 투자란 감이 아닌 숫자와 규칙으로 하는 투자다. 본 시스템은 Claude API를 두뇌로, KIS API를 실행 엔진으로 사용하여 사람의 감정 개입 없이 매일 자동으로 매수/매도 판단을 내리는 시스템을 구축한다.

2. 핵심 철학

감정 배제

공포에 팔고 탐욕에 사는 것은 인간의 본능이다. 시장이 흔들려도 사전에 정의된 규칙만 실행한다.

규칙 기반

명확한 조건이 충족될 때만 거래한다. 애매한 상황에서는 홀드(hold)가 기본값이다.

3종목 집중 원칙

하루 최대 매수 종목은 3개. 분산이 아닌 집중으로 확신 있는 종목에만 진입. 각 종목은 역할(슬롯)이 다르다.

완전 당일 청산

오버나잇을 하지 않는다. 14:50부터 익절 시작, 15:20까지 전량 강제 청산. 수익이 나더라도 다음 날의 리스크를 안고 가지 않는다.

이성적 손절

손절은 실패가 아니라 자본을 더 나은 기회로 이동시키는 의사결정이다. -5% 손절 시 원금 회복에 5.3%만 필요하지만, -30%까지 버티면 42.9%가 필요하다.

자기학습 피드백

매일 거래 결과를 자동 분석하여 진입 점수 파라미터를 튜닝한다. Claude가 오늘의 패턴을 복기하고 내일의 전략을 제안한다.

3. 3슬롯 시스템 (핵심)

매일 아침 9시, 시장이 열리면 3개의 슬롯을 채운다. 각 슬롯은 독립적이며, 역할이 다르다.

👑
leader
오늘의 주도주.
전체 시장 거래대금 1위권 + 수급(외인/기관) + 주도 섹터 대장
🚀
breakout
돌파매매.
갭업 +3% 이상 or 볼린저밴드 상단 돌파 + OBV 상승
🔄
pullback
눌림목매매.
오늘 -1%~-5% 하락 중 장중 반등 시작 + OBV 상승 + RSI ≤ 75

슬롯 독립성

슬롯 2(breakout)가 이상해도 슬롯 1과 3은 정상 매매한다. 문제가 생긴 슬롯만 교체를 탐색하고, 나머지 슬롯은 영향 없다. 포지션 청산 시 해당 슬롯은 자동으로 empty 상태가 되며, 11:30 이전이라면 새 종목 탐색을 재개한다.

7. 팀별 상세

공통 인프라 — 주도주 감지 market_leaders.py

"오늘 시장에서 돈이 가장 많이 몰리는 종목"을 실시간으로 파악한다. 우리가 스캔하는 450종목 밖에 있어도 찾아낸다.

소스: KIS 거래대금 순위 API (FHPST01730000)

        ↓ 합산 후 거래대금 내림차순

        ↓ 필터: 등락률 > 0 AND (외인 OR 기관 순매수)

        ↓
leader 슬롯 선정 우선 활용 + Claude 프롬프트 삽입

공통 인프라 — 섹터 로테이션 sector_rotation.py

오늘 어느 섹터에 돈이 몰리는지 실시간으로 추적한다. 강세 섹터(🔥) / 약세 섹터(🧊) 태그를 종목마다 부여해 Claude 판단에 반영한다.

국내 주식팀

매일 아침 09:07부터 3개의 슬롯에 종목을 채우고, 슬롯이 해제되면 재탐색한다. Gate 조건 불충족 시 Claude 호출 없이 즉시 스킵하여 API 비용을 최적화한다.

Gate 사전 체크

Gate 1리스크 레벨 ≥ 4 → 전체 스킵
Gate 2글로벌 outlook == negative → 전체 스킵
Gate 3국내 시황 점수 < -0.3 → 전체 스킵

포지션 감시

매수 후 보유 포지션을 계속 지켜보며 손절·익절·타임컷 조건 도달 시 즉시 주문을 실행한다. 청산 완료 시 해당 슬롯을 자동으로 empty로 해제하여 재탐색을 허용한다.

3중 안전망

단계방식반응 속도시스템 다운 시
1차KIS WebSocket 실시간 (H0STCNT0)tick 즉시❌ 동작 안 함
2차거래소 지정가 사전 주문즉시 (거래소)✅ 자동 체결
3차90초 폴링 백업최대 90초❌ 동작 안 함

9. DB 핵심 테이블

-- 슬롯 배정 현황 (당일 기준)
CREATE TABLE slot_assignments (
  slot        TEXT PRIMARY KEY,  -- 'leader' | 'breakout' | 'pullback'
  ticker      TEXT,  name TEXT,  signal_type TEXT,  reason TEXT,
  trade_date  DATE NOT NULL,
  status      TEXT DEFAULT 'empty',  -- 'active' | 'empty'
  assigned_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Hot List (국내 주식팀 출력)
CREATE TABLE hot_list (
  id INTEGER PRIMARY KEY,
  ticker TEXT, name TEXT, signal_type TEXT,
  volume_ratio REAL, price_change_pct REAL, rsi REAL,
  sector TEXT, reason TEXT,
  momentum_score REAL, obv_slope REAL, day_range_pos REAL,
  stoch_rsi REAL, bb_width_ratio REAL,
  trading_value INTEGER, exec_strength REAL,
  rs_daily REAL, rs_5d REAL,
  frgn_net_buy INTEGER, inst_net_buy INTEGER, atr_pct REAL,
  slot TEXT,  -- 'leader' | 'breakout' | 'pullback'
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 진입 컨텍스트 (자기학습 피드백)
CREATE TABLE trade_context (
  trade_id INTEGER PRIMARY KEY,
  ticker TEXT, trade_date DATE,
  signal_type TEXT, rsi REAL, entry_score REAL,
  momentum_score REAL, rs_daily REAL, rs_5d REAL,
  sector TEXT, exec_strength REAL, entry_hhmm TEXT
);

-- 자기학습 파라미터
CREATE TABLE strategy_params (
  param_name TEXT PRIMARY KEY,
