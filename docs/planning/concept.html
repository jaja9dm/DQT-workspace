1. 프로젝트 개요

목적

퀀트 투자란 감이 아닌 숫자와 규칙으로 하는 투자다. 본 시스템은 Claude API를 두뇌로, KIS API를 실행 엔진으로 사용하여 사람의 감정 개입 없이 매일 자동으로 매수/매도 판단을 내리는 시스템이다.

핵심 철학

감정 배제 — 공포에 팔고 탐욕에 사는 것은 인간의 본능이다. 시장이 흔들려도 사전에 정의된 규칙만 실행한다.

규칙 기반 — 명확한 조건이 충족될 때만 거래한다. 애매한 상황에서는 hold가 기본값이다.

이성적 손절 — 기회비용의 최대화 — 손절은 실패가 아니라 자본을 더 나은 기회로 이동시키는 의사결정이다. 손실 -5%에서 손절하면 원금 회복에 5.3%만 필요하지만, -30%까지 버티면 42.9%를 벌어야 원금이다.

큰 수의 법칙 — 51 대 49의 게임 — 단 한 번의 거래로 승부를 보려 하지 않는다. 올바른 확률 구조를 충분히 많이 반복하는 것이 핵심이다.

3종목 집중 원칙 — 시장이 좋든 나쁘든 하루 최대 매수 종목은 3개다. 분산이 아닌 집중으로 확신 있는 종목에만 진입한다. 각 종목은 역할(슬롯)이 다르다 — 주도주, 돌파매매, 눌림목매매.

완전 당일 청산 원칙 — 오버나잇을 하지 않는다. 모든 포지션은 당일 14:50 이후 수익 확정 또는 15:20까지 강제 청산한다.

자기학습 피드백 — 매일 거래 결과를 자동 분석하여 진입 점수 파라미터를 튜닝한다. Claude가 오늘의 패턴을 복기하고, 내일의 전략을 제안한다.

2. 핵심 거래 철학 — 슬롯 시스템

3슬롯 구조

매일 아침 9시, 시장이 열리면 3개의 슬롯을 채운다. 각 슬롯은 독립적이다.

슬롯 역할 핵심 조건
👑 leader 오늘의 주도주 거래대금 TOP30 + 외인/기관 수급 + 주도 섹터 대장 + 등락률 > 0
🚀 breakout 돌파매매 갭업 +3% 이상 OR 볼린저밴드 상단 돌파 + OBV 상승 + RSI ≥ 55
🔄 pullback 눌림목매매 당일 -0.3%~-5% 하락 중 반등 시작 + OBV 상승 + 거래량 1.5배+

슬롯별 맞춤 진입 조건 (v2.4.0 추가)

슬롯마다 역할에 맞는 추가 차단 조건이 Gate 4.5에서 적용된다.

슬롯 추가 차단 조건
👑 Leader pullback_rebound / opening_plunge_rebound 신호 차단 (눌림목 신호는 Leader에 부적합) · 일중 등락폭 위치(drp) < 0.25 차단 (당일 저점에 붙어있는 종목 차단)
🚀 Breakout drp < 0.5 차단 (일중 하단 50% 이하) · 거래량 비율 < 2.0 차단 · OBV ≤ 0 차단
🔄 Pullback drp > 0.70 차단 (intraday_pullback 신호 제외) · OBV ≤ 0 차단 · RSI > 72 차단 · MACD buy_pre가 아닌 종목 하루 종일 차단

종목 선정 유니버스 → 슬롯 할당 흐름

전체 종목 (~2,500개)
       ↓ ATR < 1.5% 필터, 거래대금 < 50억 필터
450종목 유니버스
       ↓ ThreadPoolExecutor(10) 병렬 스캔 (60일 OHLCV + 지표 계산)
       ↓ 거래량 급등 (3배+), 가격 급등 (3%+), RSI 과매수/과매도 스캔
Hot List 후보 (수십 종목)
       ↓ 신호 수 ≥2 OR 거래량 5배 급등, 거래대금 ≥ 50억
최종 Hot List (10~20종목)
       ↓ Claude 분석 — 슬롯별 조건 매칭
슬롯 배정 (leader 1 / breakout 1 / pullback 1)

공통 인프라 0-1. KIS 게이트웨이

기능 내용
토큰 관리 Access Token 발급·갱신·만료 감지. 만료 30분 전 자동 갱신
Rate Limit 초당·분당 한도 큐잉. 우선순위: TRADING > REALTIME > BACKGROUND
요청 라우팅 포지션 감시 > 매매팀 > 수집팀 순 우선 처리
장애 처리 3회 재시도 → 실패 시 전 팀 알림 + 거래 중지
모의/실전 KIS_MODE=paper|live 환경변수 1개로 전체 전환

공통 인프라 0-2. 주도주 감지 (market_leaders.py)

한 줄 목표: "오늘 시장에서 돈이 가장 많이 몰리는 종목"을 실시간으로 파악한다.

동작 방식

KOSPI 거래대금 TOP30 + KOSDAQ 거래대금 TOP30
        ↓ 합산 후 거래대금 내림차순 정렬
거래대금 TOP30 확정
        ↓ 필터: 등락률 > 0 AND (외인 OR 기관 순매수)
주도주 확정 (통과 없으면 TOP5 그대로 사용)
        ↓
5분 캐시 → Claude 프롬프트 삽입 + leader 슬롯 선정 우선 활용

공통 인프라 0-3. 섹터 로테이션 캐시 (sector_rotation.py)

한 줄 목표: 오늘 어느 섹터에 돈이 몰리는지 실시간으로 추적한다.

스캔 사이클마다 전체 종목의 등락률을 섹터별 집계
        ↓ 섹터별 평균 수익률 vs KOSPI 수익률 비교
강세 섹터 (vs_kospi > +0.5%) → 🔥 태그 → 진입점수 +5pt
약세 섹터 (vs_kospi < -0.5%) → 🧊 태그 → 진입점수 -3pt
        ↓
leader/breakout: 강세 섹터 우선 배정
position_monitor: 약세 섹터 전환 시 포지션 교체 트리거

팀 3. 매매팀 — 5단계 게이트 + 진입 점수

5단계 게이트 (순서대로)

Gate 1   — 리스크 레벨 ≥ 4 → 즉시 중단
Gate 1.5 — 슬롯 수 상한 체크: min(max_positions, risk.max_slots) 초과 시 차단
Gate 2   — 글로벌 outlook negative → 매수 보류
Gate 3   — 시장 점수 ≤ -0.5 → 매수 보류
Gate 4   — 종목별 Hard Fail 조건 체크
Gate 4.2 — 진입 신뢰도 점수 산출 → 50점 미만 차단

팀 4-1. 포지션 감시

감시 주기: 90초 (WebSocket 1차 + 거래소 지정가 2차 + 폴링 3차 3중 안전망)

핵심 성능 개선 (v2.3.0): 포지션별 MACD 조회를 N번 개별 쿼리 → preload_macd_cache() 1회 배치 쿼리로 통합.

트레일링 스톱 메커니즘

초기 손절선 = 매수가 × (1 - 2.0%)   ← ATR 기반 동적 조정 가능
                                                        ┐
수익 +3.0% 달성 → 손절선 상향 시작                      │
                                                        │
손절선은 절대 내려가지 않음 (한 방향 상승)               │ 손절선 이동
                                                        │ (floor = -2.5%)
MACD bearish or sell_prep or 14:00 이후                │
  → floor 절반(-1.25%)으로 타이트하게 조정              ┘

[v2.4.0 추가] VWAP 동적 손절선 하한:
  vwap_floor = max(매수가 × 0.98, VWAP × 0.99)
  일반 손절선보다 높으면 VWAP 기준 손절선 채택
  → 매수가 대비 -2% 이상 OR VWAP 아래 -1% 이상이면 손절

현재가 ≤ 손절선 → 전량 시장가 매도

감시 및 실행 조건

조건 동작
현재가 ≤ 손절선 전량 시장가 매도

6. DB 핵심 테이블

-- 슬롯 배정 현황 (당일 기준)
CREATE TABLE slot_assignments (
  slot        TEXT PRIMARY KEY,  -- 'leader' | 'breakout' | 'pullback'
  ticker      TEXT,
  trade_date  DATE NOT NULL,
  status      TEXT DEFAULT 'empty',  -- 'active' | 'empty'
  reason      TEXT,
  assigned_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_slot_trade_date ON slot_assignments(trade_date);

-- Hot List (국내 주식팀 출력)
CREATE TABLE hot_list (
  ticker           TEXT,    name TEXT,    signal_type TEXT,
  volume_ratio     REAL,    price_change_pct REAL,
  rsi              REAL,    sector TEXT,  reason TEXT,
  momentum_score   REAL,    obv_slope REAL,
  stoch_rsi        REAL,    bb_width_ratio REAL,
  trading_value    INTEGER, exec_strength REAL,
  rs_daily         REAL,    frgn_net_buy INTEGER, inst_net_buy INTEGER,
  atr_pct          REAL,    slot TEXT,
  created_at       DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_hot_list_ticker ON hot_list(ticker);

-- 장중 MACD 신호
CREATE TABLE intraday_macd_signal (
  ticker      TEXT,
  signal      TEXT,   -- 'buy_pre' | 'sell_pre' | 'sell_prep'
  hist_3m     REAL,   hist_5m REAL,
  sig_3m      TEXT,   sig_5m TEXT,
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_macd_signal_ticker ON intraday_macd_signal(ticker, created_at DESC);

-- 진입 컨텍스트 (자기학습)
CREATE TABLE trade_context (
  trade_id       INTEGER PRIMARY KEY,
  entry_score    REAL,    momentum_score REAL,
  rs_daily       REAL,    exec_strength REAL,
  entry_hhmm     TEXT,
  created_at     DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 자기학습 파라미터
CREATE TABLE strategy_params (
  param_name   TEXT PRIMARY KEY,
  current_val  REAL,    default_val REAL,
  updated_at   DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 섹터 강도
CREATE TABLE sector_strength (
  sector      TEXT PRIMARY KEY,
  avg_ret_1d  REAL,    vs_kospi REAL,
  updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

7. 리스크 정책

  • 3종목 상한: 하루 최대 매수 3종목 (슬롯 수와 동일)

  • 진입 시간: 09:07~12:00 범위 외 신규 매수 차단
  • 실전 전환 조건: 모의투자 3개월 + 수익률 플러스 확인
